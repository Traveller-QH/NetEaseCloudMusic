# 登录页面调试指南

## 问题描述
浏览器控制台报错：
```
TypeError: Cannot read properties of undefined (reading 'isLogin')
```

## 问题原因
访问 userStore 属性的方式错误，应该是 `userStore.isLogin` 而不是 `userStore.state.isLogin`

## 修复记录

### 第一次修复 (my.vue)
```javascript
// 错误写法
const isLogin = computed(() => userStore.state.isLogin)

// 正确写法
const isLogin = computed(() => userStore.isLogin)
```

### 第二次修复 (Sidebar.vue)
```javascript
// 错误写法
const isLogin = computed(() => userStore.state.isLogin)

// 正确写法
const isLogin = computed(() => userStore.isLogin)
```

### 第四次修复 (App.vue)
```javascript
// 错误写法
const restored = userStore.restoreUserInfo()
userStore.checkLoginStatus()

// 正确写法
const restored = userStore.restoreLoginState()
userStore.validateLoginStatus()
```

## 常见问题排查

### Q: 页面仍然报错怎么办？
A: 检查以下几点：
1. 确认 userStore 正确导入
2. 检查 Pinia 是否正确初始化
3. 查看控制台完整错误信息
4. 确认所有依赖文件存在

### Q: 登录后数据不显示怎么办？
A: 检查以下几点：
1. 确认登录接口调用成功
2. 检查 userStore 状态是否正确更新
3. 查看本地存储是否有数据
4. 确认页面计算属性是否正确

### Q: 页面刷新后登录状态丢失怎么办？
A: 检查以下几点：
1. 确认 `restoreLoginState` 方法正确实现
2. 检查本地存储读写是否正常
3. 确认 cookie 和 token 的持久化
4. 验证登录状态验证逻辑
